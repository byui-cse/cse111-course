CSE 111 | Programming with Functions

03 Prepare: Calling Functions

Because most useful computer programs are very large, programmers divide their programs into parts. Dividing a program into parts makes it easier to write, debug, and understand the program. A programmer can divide a Python program into modules, classes, and functions. In this unit, you will learn how to use existing functions, and in the next unit, you will learn how to write your own functions.

Concepts

Here are the Python programming concepts and topics that you should learn during this lesson:

  1. A function is a group of statements (computer commands) that together perform one task.
  2. Python includes many built-in functions such as: input, print, open, len, range, int, float, str, abs, round, list, set, and dict. You can read about the built-in functions in the Built-in Functions section of the official Python online reference.
  3. A Python module is collection of related functions.
  4. Python includes many standard modules which have more functions, such as the math module which includes the floor, ceil, and sqrt functions and the csv module which includes the reader and writer functions. You can read more about the standard modules in the Python Module Index.
  5. A programmer (you) can save lots of time by using existing functions.
  6. A programmer uses a function by calling it (also known as invoking it). To call (or invoke) a function means to write code that causes the computer to execute the code that is inside of that function. During CSE 110, you often wrote code that called the input and print functions. For example: 
    name = input("Please enter your name: ")
print(f"Hello {name}")
  7. To correctly call a function you must know the following:
    1. The name of the module in which the function is located
    2. The name of the function
    3. The parameters that the function takes
    4. What the function does
    All of this information is normally available in online documentation. For example, from the online Python documentation about the math module, we read this about the sqrt function:
    math.sqrt(x)
    Return the square root of x.
    From this short description, we know the following:
    1. The name of the containing module is math
    2. The name of the function is sqrt
    3. The function takes one parameter named x
    4. The function computes and returns the square root of a number
  8. A parameter is a piece of data that a function needs in order to complete its task. In the online documentation for the math module, we see that the math.sqrt function has one parameter named x which is the number that math.sqrt will compute the square root of.
    math.sqrt(x)
    Return the square root of x.
    In order for the math.sqrt function to do its job (compute the square root), a programmer must write code to pass a number through the parameter x into the math.sqrt function. Here is a simple example of passing a number to the math.sqrt function: 
    r = math.sqrt(71)
    In the above example, 71 will be passed through the parameter x into the math.sqrt function. The math.sqrt function will compute the square root of 71 and return the computed value which will then be stored in the variable r.
  9. An argument is the value that is passed through a parameter into a function. In other words, parameters are listed in a function's header, and arguments are listed in a call to a function. In this example code: 
    r = math.sqrt(71)
    71 is the argument that is passed into the math.sqrt function through its parameter x. When a function has more than one parameter and a programmer writes code to call that function, the programmer usually writes the arguments in the same order as the parameters. Consider the built-in Python round function:
    round(number, ndigits)
    Return number rounded to ndigits precision after the decimal point. If ndigits is omitted or is None, round returns the nearest integer to number.
    Now consider this Python code that gets a number from a user, rounds that number to two digits after the decimal, and then prints the rounded number. 
    1
2
3
    n = float(input("Please enter a number: "))
r = round(n, 2)
print(r)
    In the previous example,
    • The code on line 1 calls both the input and float functions.
    • Line 2 calls the built-in round function and passes two arguments. Notice that the order of the arguments matches the order of the parameters. Specifically the number to be rounded (n) is the first argument and the number of digits after the decimal point (2) is the second argument.
    • Line 3 prints the rounded number.
  10. When calling a function, some arguments are optional. Again consider the description of the built-in round function:
    round(number, ndigits)
    Return number rounded to ndigits precision after the decimal point. If ndigits is omitted or is None, round returns the nearest integer to number.
    From the description we read that the second argument is optional. If the programmer doesn't pass a second argument, the number will be rounded to an integer.
  11. For some optional arguments, we must pass a named argument which is an argument that is preceded by the name of its matching parameter. For example, here is an excerpt from the documentation for the print function:
    print(*objects, sep=' ', end='\n', file=sys.stdout, flush=False)
    Print objects to the text stream file, separated by sep and followed by end. sep, end, file and flush, if present, must be given as keyword arguments.
    Notice from the excerpt that the print function can take many objects that will be printed. Optionally, it can take parameters named sep, end, file, and flush which must be named when they are used. For example, this code calls the print function to print three words all separated by a vertical bar (|). Notice the named argument sep. 
    x = "sun"
y = "moon"
z = "stars"
print(x, y, z, sep="|")
