makehtml — Write U.S. Weather forecasts to an HTML File

The makehtml module writes U.S. National Weather Service forecasts to an HTML file.

To successfully use this module to write an HTML file, your code must first retrieve the weather forecasts from the U.S. National Weather Service. The weather service organizes the weather forecast into periods of time. Each period covers about 12 hours and is either a day or a night.

After your code retrives the periods, it should call these functions from this module:

  1. makehtml.create_report(filename, locname, options)
    Creates and returns a weather report that will be used for adding rows and periods and then writing the report. create_report must be called once before the other functions.
    param filename: path and name of the HTML file where this module will write the weather forecasts.
    param locname: name of the geographic location (city) where the forecasts apply.
    param options: an optional list of weather forecast data that the user desires to be written in a report.
  2. makehtml.add_row(report)
    Adds one row of periods to the weather report. add_row must be called once at the beginning of each desired row of periods.
  3. makehtml.add_period(report, period)
    Adds one forecast for a period to the current row in the weather report. add_period must be called once for each period in the current row.
    param report: the report to add the period to
    param period: a forecast for a period of time that came from the U.S. National Weather Service
  4. makehtml.write_report(report)
    Writes the weather forecasts to an HTML file. write_report must be called once, after your code has added all desired rows and periods.
