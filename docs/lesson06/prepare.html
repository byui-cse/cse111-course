Variable Values and References

In Python, variables are processed differently based on their data type. For example, here is a small Python program that contains two integer variables named x and y.

def main():
    x = 3
    y = x
    print(f"x: {x}  y: {y}")
    y += 1
    print(f"x: {x}  y: {y}")

main()

The previous code will print this:

> python prep1.py
x: 3  y: 3
x: 3  y: 4

Within the previous code,

  • the statement on line 2 stores the value 3 into the variable x.
  • Line 3 copies the value that is in the variable x into the variable y.
  • Line 4 prints the values of x and y which are both 3.
  • Line 5 adds one to the value of y, making its value 4 instead of 3.
  • Line 6 prints the values of x and y. The value of x remained unchanged. The value of y was changed to 4.

Why did line 5 (y += 1) change the value of y but not change the value of x? Because line 3 copied the value that was in x into y.

def main():
    lx = [7, -2]
    ly = lx
    print(f"lx: {lx}  y: {ly}")
    ly.append(5)
    print(f"lx: {lx}  y: {ly}")

main()

Here is the output of the second program.

> python prep2.py
lx: [7, -2]  y: [7, -2]
lx: [7, -2, 5]  y: [7, -2, 5]

From the output of this second program, we can see there is a big difference between the way a Python program assigns integers and the way it assigns lists. Within the second program,

  • the statement on line 2 creates a list and stores a reference to that list in the variable lx.
  • Line 3 copies the reference in the variable lx into the variable ly. Line 3 does not create a copy of the list but instead causes both the variables lx and ly to refer to the same list.
  • Line 4 prints the values of lx and ly. Notice that their values are the same as we expect them to be because of line 3.
  • Line 5 appends the number 5 onto the list ly.
  • Line 6 prints the values of lx and ly again. Notice in the output that when lx and ly are printed the second time, it appears that the number 5 was appended to both lists.

Why does it appear that appending the number 5 onto ly also appended the number 5 onto lx? What really happened is that lx and ly refer to the same list. There is really only one list with two references to that list. Because lx and ly refer to the same list, a change to the list through variable ly can be seen through variable lx.

When a computer executes a Python statement to assign the value of a boolean, integer, or float variable to another variable (y = x), the computer copies the value of one variable into the other. However, when a computer executes a Python statement to assign the value of a string, list, set, or dictionary variable to another variable (ly = lx), the computer does not copy the value but instead copies the reference so that both variables refer to the same value in memory.

The fact that the computer copies the value of some data types (boolean, integer, float) and copies the reference for other data types (list, set, dict) has important implications for passing arguments into functions. Consider this Python program with two functions named modify_args and main.

def modify_args(n, ls):
    """Demonstrate that the computer passes a value
    for integers and passes a reference for lists.
    """
    print("    modify_args()")
    print(f"   n: {n}  ls: {ls}")

    # Change the values of both parameters.
    n += 1
    ls.append(5)

    print(f"   n: {n}  ls: {ls}")


def main():
    print("main()")
    x = 3
    lx = [7, -2]
    print(f"x: {x}  lx: {lx}")

    # Pass one integer and one list
    # to the modify_args function.
    modify_args(x, lx)

    print(f"x: {x}  lx: {lx}")


# Call main to begin this program.
main()

Within this third program,

  • the statement on line 17 assigns the value 3 to a variable named x.
  • Line 18 assigns a list to a variable named lx.
  • Line 19 prints the values of x and lx before they are passed to the modify_args function.
  • Line 23 calls the modify_args function and passes x and lx to that function.
  • Within the modify_args function, line 7 changes the value of the parameter n by adding one to it.
  • Line 25 prints the values of x and lx after they were passed to the modify_args function. Notice in the output below that the value of x was not changed by the modify_args function. However, the value of lx was changed by the modify_args function.
> python prep3.py
main()
x: 3  lx: [7, -2]
    modify_args()
    n: 3  ls: [7, -2]
    n: 4  ls: [7, -2, 5]
x: 3  lx: [7, -2, 5]

From the output of the third program, we can see that modifying an integer parameter changes the integer within the called function only. However modifying a list parameter changes the list within the called function and the calling function. Why? Because when a computer passes a boolean, integer, or float variable to a function, the computer copies the value of that variable into the parameter of the called function. However, when a computer passes a list, set, or dictionary variable to a function, the computer copies the reference so that the original variable and the parameter both refer to the same value in memory.
