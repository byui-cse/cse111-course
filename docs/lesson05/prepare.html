05 Prepare: Writing Functions

Throughout your programming experience, you've been using functions. A function is a group of statements that together perform one task.

Types of Functions

Broadly speaking, there are four types of functions you'll come across in python:

  1. Built-in functions

  2. Standard library functions

  3. Third-party functions

  4. User defined functions

Regardless of what kind of functions you're using, they are called by using their name followed by a set of ()'s.

Note that sometimes people refer to calling a function as invoking the function or executing the function.

1. Built-in Functions

Python includes many built-in functions. Some examples of built-in functions you've been using include:

  • print() - A function that displays output to the user.

  • input() - A function that gets input from the user.

  • int() - A function that converts a value to an integer.

  • float() - A function that converts a value to a float.

These are called "built-in" functions because you don't have to import any modules to use them. They are just part of the Python language.

# Calling some built-in functions
print("Hello!")
age = input("What's your name? ")
nextYear = int(age) + 1

2. Standard Library Functions

You've also used a number of standard library functions. These are functions that come with Python, but require you to import a library or module to use. Grouping similar functions together in a library, helps to keep things organized.

Some examples you've used include:

  • math.round() - A function in the math module that rounds a number.

  • string.upper() - A function that converts a string to uppercase.

  • list.append() - A function that adds an item to a list.

  • random.randint() - A function that gives you a random number.

Notice that some of these functions are called by using the module name where the function is defined. Such as the round() function:

import math

# Round the number to 2 decimal places by 
# calling the round() function in the math module.
answer = math.round(1.2312344, 2)

Sometimes a function will belong to a particular data type.

For example, the string data type contains a number of functions. When a variable contains a string, you can access its functions, using this syntax: variable.functionName().

Functions that belong to a particular data type are often referred to as methods. Methods tend to act directly on the object they belong to. For example the string.upper() method returns the uppercase equivalent of whatever string it's attached to:

choice = input("What would you like to do? ")

# Call the upper() method attached to the string contained
# in the choice variable, in order to convert that string to
# uppercase.
if choice.upper() == "LEARN":
    print("That's a good choice.")

3. Third-party Functions

Sometimes, you'll want your code to do something that isn't easily accomplished using only the built-in or standard library functions. In those cases, programmers often install third party modules (sometimes called "libraries"), in order to use additional functions that others have written.

For example, the Pillow module is a module you may have used in CSE 110. This module provides several functions for working with images.

These libraries are usually installed using the pip tool. Once a library is installed, you can import it and use its functions just like any standard library function.

4. User-defined Functions

Often, programmers will write their own functions. Creating these user-defined functions provides several advantages:

  • It makes your code more reusable.

  • It makes your code easier to understand and debug.

  • It makes it easier for you to make changes or add capabilities to your code.

In this unit, you'll learn how to create your own user-defined functions in Python.

Defining a Function

All function definitions start with the keyword def (which stands for "define"), followed by the name of the function.

Function definitions always end with a set of ()'s, followed by a :.

You could read the following line of code as, "Define a function called sayHello."

def sayHello():

Just like with other block level statements you've seen, such as if, else, and for, (all of which also use :'s), the statements inside the body of a function are indented:

# Define a function called "sayHello"
def sayHello():
    # This is the "body" of the function.
    # When the sayHello function is "called", this is
    # the code that will execute:
    print("Hello!")
    print("Hope you have a good day!")

The concept of defining a function is important to remember. When we define a function, we're essentially saying:

Okay python, you know what functions like print and input mean, but I'm going to teach you a new function called sayHello. Whenever you see that function written down, I want you to execute the following statements...

Calling a Function

You've been calling functions since you wrote your first print() statement. Calling user-defined functions works the same way. Once you've defined a function, you can call it using the function name followed by a set of ()'s:

# Define the function
def sayHello():
    print("Hello")
    print("Hope you have a good day!")

# Call the function
sayHello()

Function Parameters

Just as with other functions you've used, user-defined functions can make use of parameters.

When you define a function, you specify its parameters inside of the ()'s.

The code inside the body of the function can use those parameters like any other variable:

# Define a function that uses parameters.

# Whatever argument is passed to the function 
# when it is called, will be stored in the "numbers" 
# parameter, and will be accessible within the 
# body of the function.
def printAverage(numbers):
    total = 0
    for number in numbers:
        total += number

    average = total / len(numbers)
    print(f"The average is: {average}")

Calling a user defined function that requires parameters works exactly like other functions you've used:

scores = [100, 90, 85, 84, 99]

# Call the printAverage function and pass the scores
# list as an argument. 
printAverage(scores)

Function Return Values

Many functions you've used in the past return a value. When a function returns a value, we usually store that value in a variable to use later.

Here's a new version of the printAverage function that returns the average rather than printing it. Having a function return a value, instead of printing it, makes our functions more reusable.

In the example below, notice that we changed the name of the function to calculateAverage instead of printAverage.

Functions should always be named so that it's obvious what they do. Well-named functions almost always start with a verb.

# Define a function that uses parameters.
def calculateAverage(numbers):
    total = 0
    for number in numbers:
        total += number

    average = total / len(numbers)

    # The value we return will be available 
    # wherever the function was called from.
    return average


# Call the function and store its return value
# in a variable to use later.
scores = [100, 90, 85, 84, 99]
finalGrade = calculateAverage(scores)

While it's usually a good practice, you don't have to store the return value of a function in a variable. Sometimes you'll see it used directly like in the following examples:

scores = [100, 90, 85, 84, 99]

# Call the function and immediately print its return value:
print("The average is: ")
print( calculateAverage(scores) )

# Here we hold on to the return value just long enough to 
# ask the question in the if statement:
if calculateAverage(scores) >= 93:
    print("You got an A ðŸ¤©")
else:
    print("You did not get an A ðŸ˜ž")

Notice that in the above example, we're calling the function twice. Once when we want to print the average, and again when we want to see if got an A or not. Every time you call a function, the code inside the function body is executed.

In this case, since our argument is the same list of scores both times, the result will be the same in both cases. So it would be faster to save the result in a variable and reuse the variable instead:

scores = [100, 90, 85, 84, 99]

# Call the function and store the result so 
# we can use it later.
finalGrade = calculateAverage(scores)

print("The average is: ")
print(finalGrade)

if finalGrade >= 93:
    print("You got an A ðŸ¤©")
else:
    print("You did not get an A ðŸ˜ž")
