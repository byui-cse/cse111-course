07 Checkpoint: Testing Code

Overview

Complete this individual checkpoint assignment to demonstrate your ability to verify the correctness of functions by writing test functions and using pytest.

Helpful Documentation

pytest is well documented at docs.pytest.org/en/latest/getting-started.html. However, not all of the documenation about pytest applies to your tasks in CSE 111. The following parts of that documentation do apply and may help you as you complete this lesson's assignments.

  • Create your first test
  • Calling pytest through python -m pytest

Assignment

Write two test functions and run them with pytest.

  1. Download this Python file: words.py, and save it in the same folder where you will save your Python program. Then open the downloaded file in VS Code.
  2. Notice that downloaded file contains two small functions named: prefix and suffix. Notice also that each function has a documentation string that describes what the function does. Read the documentation strings for both functions.
  3. On the menu bar for VS Code, click "Terminal" and then "New Terminal". This will open a terminal frame at the bottom of the VS Code window. A terminal is a window or frame where a user can type and execute computer commands.
  4. Copy and paste the following command into the terminal frame and execute the command by pressing the Enter key. 
    python -m pip install --user --upgrade pip setuptools wheel
    That command will upgrade pip and several other parts of the Python installation modules so that pip will work correctly.
  5. Install the pytest module by copying, pasting, and executing the following command in the terminal frame. 
    python -m pip install --user pytest
  6. In VS Code open a new Python file and copy and paste this import statement at the type of the file: 
    from words import prefix, suffix
  7. Write a function named test_prefix that takes no parameters and contains assert statements for these nine calls to prefix:
    Parameters Return
    Value
    s1 s2
    "" "" ""
    "" "correct" ""
    "clear" "" ""
    "happy" "funny" ""
    "cat" "catalog" "cat"
    "dogmatic" "dog" "dog"
    "jump" "joyous" "j"
    "unwise" "ungrateful" "un"
    "Disable" "dIstasteful" "dis"
  8. Write a function named test_suffix that takes no parameters and contains assert statements for these nine calls to suffix:
    Parameters Return
    Value
    s1 s2
    "" "" ""
    "" "correct" ""
    "clear" "" ""
    "angelic" "awesome" ""
    "found" "profound" "found"
    "ditch" "itch" "itch"
    "happy" "funny" "y"
    "tired" "fatigued" "ed"
    "swimming" "FLYING" "ing"
  9. Save your Python file.
  10. In the terminal frame in VS Code, run your test functions by copying, pasting, and executing this command: 
    python -m pytest <your_python_filename>.py
