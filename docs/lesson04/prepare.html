CSE 111 | Programming with Functions

04 Prepare: Calling Third-Party Functions

Because most useful computer programs are very large, programmers divide their programs into parts. Dividing a program into parts makes it easier to write, debug, and understand the program. A programmer can divide a Python program into modules, classes, and functions. In this lesson, you will learn how to call existing functions in third-party modules, and in the next unit, you will learn how to write your own functions.

Concepts

Here are the Python programming concepts and topics that you should learn during this lesson:

  1. A function is a group of statements (computer commands) that together perform one task.
  2. A Python module is collection of related functions.
  3. Python includes many standard modules, such as csv, datetime, math, random, and textwrap. You can read the documentation for the standard modules in the official Python reference.
  4. pip is a standard Python module that you can use to download and install third-party modules. During this course, you will use pip to download and install several third-party modules.
  5. There are many modules that are not part of the standard Python modules. These additional modules are sometimes called third-party modules and are written and maintained by other people as a service to the Python community. Third-party modules are usually available at no cost.
  6. The Python package index (pypi) is an online repository of software that contains more than 200,000 software packages, each of which contains one or more third-party Python modules.
  7. The requests module contains functions to send HTTP requests for data to a computer server on the internet and to read the data that the server returns.
  8. A programmer (you) can save lots of time by using existing modules.

Documentation

To learn more about installing modules using the pip command, read the Basic usage section of the official pip reference.
