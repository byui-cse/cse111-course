04 Team Activity: Retrieve Star Wars Data from a Web Server

Overview

Arrange a one-hour synchronous meeting with your team for this activity. Online students should coordinate a video-sharing meeting. Campus students will use class time for this meeting. You should prepare for this meeting by completing the preparation material and the individual checkpoint assignment beforehand.

Problem Statement

Star Wars is a series of movies about fictional characters and events that lived and happened a long time ago in a galaxy far far away. Because the movies are popular, there are many fan websites that contain information about the movies. One such website is swapi.co swapi.co maintains a web service that contains lots of data gleaned from the movies and books about the fictional characters and places that are part of the Star Wars galaxy. During this activity, your team will write a Python program that uses the requests module to retrieve data from the swapi service.

Helpful Documentation

The Python requests module contains functions to send http requests for data to a computer server on the internet and to read the data that the server returns. Read about the requests module at requests.readthedocs.io

The Star Wars API (swapi) maintains lots of data about the ficitional Star Wars galaxy. Read about swapi and experiment with it at swapi.co/documentation

Instructions

Open the Star Wars API documenation in your browser. From the documentation you can read that this URL https://swapi.co/api/planets will return data about all the planets in the Star Wars galaxy, and this URL https://swapi.co/api/planets/{i} where {i} is an integer will return data about an individual planet.

Use the commodities.py program, that you downloaded during this lesson's checkpoint, as a reference, and write a Python program to retrieve the population for each planet and calculate and display the total population of all planets in the Star Wars galaxy.

Core Requirements

  1. Your program must use the requests module to retrieve data from this URL: https://swapi.co/api/planets and then get the count of planets from that data.
  2. Your program must contain a loop that repeats once for each planet. Within that loop your program must use the requests module to retrieve the data for each planet and then get the population from that data and add that population into a total population.
  3. Your program must print the total population.

Stretch Challenges

If your team finishes the core requirements in less than an hour, complete one or more of these stretch challenges. Note that the stretch challenges are optional.

  1. Add something or change something in your program that you think would make your program better, easier for the user, more elegant, or more fun. Be creative.
  2. Write comments above the lines of code that were difficult for you to understand.
  3. Change your program to count the number of planets that are known to have surface water.
  4. Change your program to compute the average height for all the characters in the swapi database.

Warning

Now that you know how to use the requests module to retrieve data, you have enough knowledge to launch a denial of service (DOS) attack against a web server. If you write a program that quickly makes thousands or millions of requests to a server, your program's requests will overwhelm the server. Then other programs that are making normal, legitimate requests will not be served because the server is too busy with the flood of requests from your program. This is a denial of service attack. Don't do this. Launching a DOS attack is rude, unethical, likely illegal, a violation of the BYU-Idaho honor code, and not something that a disciple of Jesus Christ would knowingly do. Launching a DOS attack, either purposefully or accidentally, will lead to your computer's IP address being locked on the server which means the server will refuse to serve your computer. With great power, comes great responsibility.
