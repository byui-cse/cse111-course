CSE 111 | Programming with Functions

08 Checkpoint: Commodities

Overview

Complete this individual checkpoint assignment to demonstrate your ability to write code that handles errors that may occur while your program is running.

Helpful Documentation

In the Error and Exceptions section of the requests documentation, read the list of exceptions that the requests module may raise.

Assignment

Add error handling to the commodities.py program from lesson 4 by doing the following:

  1. Download and save the commodities.py program and then open it in VS Code.
  2. Copy the API key in I-Learn in the page that corresponds to this activity and paste it in place of the string, "The API key is in your I-Learn course."
  3. Move all of the code except the import statements into a function named main.
    1. Use your mouse to select all of the code from line 4 to the end of the file.
    2. Press the "Tab" key on your keyboard. This will indent all of the code that you selected.
    3. Click on line 3 to unselect all of the code. Then press the "Enter" key which will add a new blank line.
    4. On the new blank line, type the header for the main function: def main():
  4. At the very bottom of the file, add two blank lines.
  5. On the last blank line, type a call to the main function: main()
  6. Move all of the code in the main function into a try block.
    1. Use your mouse to select again all of the code that is in the main function.
    2. Press the "Tab" key on your keyboard. This will indent all of the code that you selected.
    3. Immediately beneath the header for the main function, add a blank line.
    4. On the blank line, type a try header like this: 
      try:
  7. At the bottom of the main function, add an exception block for each of the five exceptions that the requests module may raise. Also add an exception block for a generic Exception.
  8. Within each exception block, add a statement that prints the name of the exception.

Testing Procedure

Verify that your program works correctly by following each step in this testing procedure:

  1. Run your program. If your program contains syntax errors, fix the errors, so that your program runs correctly. Your program should run without errors and print the price of orange juice in the United States for the previous 13 months.
  2. Disconnect your computer from the network by either disconnecting the network cable or turning off your computer's WiFi adapter.
  3. With your computer disconnected from the network, run your program again. Your program should print "Connection error" to the console.
