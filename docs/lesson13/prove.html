CSE 111 | Programming with Functions

13 Prove Assignment: Significant Python Program

Overview

Prove that you can write a significant Python program that uses real world data and solves a real world problem. Your program must include multiple functions that you verify are correct with test functions and pytest.

Assignment

Finish developing the program that you started in the prove assignment for lesson 11.

Submission

At the end of this lesson, you must complete this lesson's Prove: Assignment quiz in I-Learn. Your teacher or teaching assistant will grade your work according to this rubric:

  1. Time - 50%: Did you spend at least six hours (including class time for face-to-face students) on your program during the current lesson?
  2. Description - 10%: Is the description of your work complete and easily understandable? At the end of each lesson, you will submit a description of your work. This description should follow normal English rules of spelling and grammar. It must include the following:
    1. The amount of time that you spent working on your program during the current lesson.
    2. A list of the function names in your program.
    3. A list of the test function names in your test code.
    4. A list of the documentation that you read, the videos that you watched, and the coding experiments that you tried.
    5. A list of the work that you finished on your program.
  3. Program - 24%
    1. Is your program divided into functions and does each function perform one task only?
      • Some attempt made - 1 point
      • Developing but significantly deficient - 2 points
      • Slightly deficient - 3 points
      • Meets expectations - 4 points
    2. Does your program effectively uses existing Python modules such as math, random, requests, pandas, and tkinter?
      • Some attempt made - 1 point
      • Developing but significantly deficient - 2 points
      • Slightly deficient - 3 points
      • Meets expectations - 4 points
    3. Does your program perform a significant real world task?
      • Some attempt made - 1 point
      • Developing but significantly deficient - 2 points
      • Slightly deficient - 3 points
      • Meets expectations - 4 points
  4. Test Functions - 16%
    1. Each testable program function is covered (tested) by one test function. (Some functions, especially main and those that create GUIs are difficult to test and don't need to have a corresponding test function.)
      • Some attempt made - 1 point
      • Developing but significantly deficient - 2 points
      • Slightly deficient - 3 points
      • Meets expectations - 4 points
    2. Each test function completely exercises (tests) its corresponding program function. In other words, the test function calls the program function multiple times with different arguments, including unusual or unexpected values.
      • Some attempt made - 1 point
      • Developing but significantly deficient - 2 points
      • Slightly deficient - 3 points
      • Meets expectations - 4 points
