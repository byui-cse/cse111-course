Problem Statement

A common task for many knowledge workers is to use a number, key, or ID to look up information about a person. For example, a knowledge worker may use a phone number as a key to find (or look up) information about a customer. During this activity, your team will write a Python program that uses a student's I-Number to look up the student's name.

Helpful Documentation

A csv file is a text file that contains a list of data. Within each row of a csv file, the data items are separated by a character which is usually a comma. The name "csv" is an acronym for "comma separated values." Python includes functions for processing csv files. Read about the Python csv module in one of these articles:

A Python dictionary is a collection of data that is stored in pairs of keys and values. The keys are used to store and quickly find the values. Read about Python dictionaries in one of these articles:

Assignment

Download this file: students.csv and save it in the same folder where you will save your Python program. Open the file in VS Code or another text editor and examine it. Notice that the I-Numbers and names are separated by a comma. Notice also that the I-Numbers are stored in the file without any dashes between the digits.

Write a Python program that reads the contents of the students.csv file and populates a Python dictionary while reading the file. The dictionary uses student I-Numbers as the keys and student names as the values. After your program populates the dictionary from the file, your program must ask the user for an I-Number. Your program will then find and print the corresponding student's name. Recall that I-Numbers contain nine digits and are often typed by users with dashes separating some of the digits like this 62-560-1400. However, the I-Numbers in the students.csv file don't contain dashes.

Core Requirements

  1. Your program must read the data in the students.csv file and convert that data into a Python dictionary with I-Numbers as the keys and names as the values.
  2. Your program must ask the user for an I-Number and must display the student name that corresponds to the given I-Number.
  3. If a user enters an I-Number that doesn't exist in the dictionary, your program must print the message, "No such student" (without the quotes).

Stretch Challenges

If your team finishes the core requirements in less than an hour, complete one or more of these stretch challenges. Note that the stretch challenges are optional.

  1. Add something or change something in your program that you think would make your program better, easier for the user, more elegant, or more fun. Be creative.
  2. Write comments above the lines of code that were difficult for you to understand.
  3. Add code to remove dashes from the I-Number that the user enters. This will allow the user to enter I-Numbers with dashes and still allow the computer to search in the dictionary.
  4. When a user enters an I-Number, your program should ensure it is a valid I-Number.
    1. If there are too few digits in the I-Number, your program should print, "Invalid I-Number: too few digits" (without the quotes).
    2. If there are too many digits in the I-Number, your program should print, "Invalid I-Number: too many digits" (without the quotes).
    3. If the given I-Number contains any characters besides digits and dashes, your program should output "Invalid I-Number" (without the quotes).
    4. Write a loop around the code that gets the user input. If the user enters an invalid I-Number, the loop should repeat so that the user is again prompted to enter an I-Number.

Testing Procedure

Verify that your program works correctly by following each step in this testing procedure:

  1. Run your program using the input shown in the sample run section above. Ensure that your program's output matches the sample run output.
  2. Run your program and give this I-Number as input: 00-115-2306 (including the dashes). Many users will want to enter I-Numbers with dashes. How should your program handle the dashes?
  3. Run your program and give an I-Number with too few digits or too many digits. How should your program handle these invalid I-Numbers?
