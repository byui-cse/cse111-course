02 Team Activity: CSV Files and Lists

Overview

Arrange a one-hour synchronous meeting with your team for this activity. Online students should coordinate a video-sharing meeting. Campus students will use class time for this meeting.

You should prepare for this meeting by completing the preparation material and the individual checkpoint assignment beforehand.

Problem Statement

A common task for many knowledge workers is to use a number, key, or ID to look up information about a person. For example, a knowledge worker may use a phone number as a key to look up information about a customer. During this activity, your team will write a Python program that uses a student's I-Number to look up the student's name.

Assignment

Download this file: students.csv and save it in the same folder where you will save your Python program.

Write a Python program that reads the contents of the students.csv file and creates two lists while reading the file. One list will contain student I-Numbers, and the other list will contain student names. After your program creates both lists from the file, your program must ask the user for an I-Number. Your program will then find the index of that I-Number in the first list and use that index to print the corresponding student name. Recall that I-Numbers contain nine digits and are correctly formatted as xx-xxx-xxxx, where each x represents one digit.

Core Requirements

  1. Your program must read the data in the students.csv CSV file and convert that data into two Python lists of strings.
  2. Your program must ask the user for an I-Number.
  3. Your program must display the student name that corresponds to the given I-Number.
  4. If a user enters an I-Number that doesn't exist in the list of I-Numbers, your program must print the message, "No such student" (without the quotes).

Stretch Challenges

If your team finishes the core requirements in less than an hour, complete one or more of these stretch challenges. Note that the stretch challenges are optional.

  1. Add something or change something in your program that you think would make your program better, easier for the user, more elegant, or more fun. Be creative.
  2. Write comments above the lines of code that were difficult for you to understand.
  3. When a user enters an I-Number, your program should ensure it is a valid I-Number.
    1. If there are too few digits in the I-Number, your program should print, "Invalid I-Number: too few digits" (without the quotes).
    2. If there are too many digits in the I-Number, your program should print, "Invalid I-Number: too many digits" (without the quotes).
    3. If the given I-Number contains any characters besides digits and dashes, your program should output "Invalid I-Number" (without the quotes).
    4. Write a loop around the code that gets the user input. If the user enters an invalid I-Number, the loop should repeat so that the user is again prompted to enter an I-Number.
