Problem Statement

In our modern world economy, many items are manufactured in large factories, then packed in boxes and shipped to distribution centers and retail stores. A common question for employees who pack items is "How many boxes do we need?"

While entering input for a program, users occasionally make mistakes and enter invalid or incorrect input. A good program recovers from user mistakes, informs users about mistakes in clear polite language, and gives users an opportunity to correct their mistakes.

Assignment

A manufacturing company needs a program that will help its employees pack manufactured items into boxes for shipping. Write a Python program named boxes.py that asks the user for two integers: 1) the number of items and 2) the items per box. Your program must require the user to enter positive integers and then must compute and print the number of boxes necessary to hold the items.

Order of Development

A good way to write this program is to first write simple code to get the two integers from the user, compute the number of boxes, and print the number of boxes. Then second, add code to validate the user input.

Helpful Documentation

The math.ceil() function rounds a number up to the nearest integer that is greater than or equal to a number.

The code example in this lesson's prepare content shows how to require a user to enter a non-negative integer between a minimum and maximum value.

Core Requirements

Your program must do the following:

  1. Compute and print the number of boxes required to hold the items. This must be a whole number. Note that the last box may be packed with fewer items than the other boxes.
  2. Prompt the user for the number of manufactured items and ensure that the user enters an integer greater than 0. If the user enters anything that is not an integer greater than 0, your program must repeatedly prompt the user to enter another integer until he enters an integer greater than zero.
  3. Prompt the user for the number of items that he will pack per box and ensure that the user enters an integer greater than 0 and less than the number of manufactured items. If the user enters anything that is not an integer greater than 0 and less than the number of manufactured items, your program must repeatedly prompt the user to enter another integer until he enters a correct integer.

Stretch Challenges

If your team finishes the core requirements in less than an hour, complete one or more of these stretch challenges. Note that the stretch challenges are optional.

  1. Review your code and ask yourself if any section of the code seems similar to another section. Write down or record the starting and ending line number of each similar section. Perhaps one of the similar sections could be separated from the other code and reused multiple times which would cause the program to have fewer lines of code.
  2. Discuss with your team how you could write code to validate that a user correctly entered any number, including negative integers and fractional numbers.
